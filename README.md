## Overview

Roshan System is an experimental, desktop-like environment and collection of small demo applications. The repository contains a primary Windows desktop implementation built with C# and WPF (under `New Roshan System/Roshan System`) and older Python/Tkinter prototype(s) under `Roshan System - Python Version` and other experimental folders.

The WPF app provides a simulated desktop with a Start menu and multiple built-in apps (Notepad, Calculator, Browser view, Overclocker UI, Run dialog, Pong demo, winver/about, and a simple mod/plugin UI).

---

## Features

- Desktop-like main window and Start menu
- Notepad with open/save and common keyboard shortcuts
- Basic Calculator
- Embedded browser view for loading URLs
- Run dialog for launching internal windows or external executables
- Overclocker demo UI (numeric adjuster)
- Pong demo and other small app demos
- Simple mod system UI (IMod interface, ModLoader, ModList, ModDetails)
- Python/Tkinter prototype included for experimentation

## Build & run

### New Roshan System (C# / WPF)
1. Open `New Roshan System/Roshan System/Roshan System.sln` in Visual Studio.
2. Restore any NuGet packages and build the solution.
3. Run from Visual Studio (Start Debugging) or run the output executable.

Notes:
- If the project targets a specific .NET Framework or SDK, install the matching SDK/workload.
- The UI references large image assets in `textures/`; missing files may break the UI.

### Python prototype
1. Open the `Roshan System - Python Version` folder and locate `main.py`.
2. Run: `python "main.py"`
3. Run from a terminal to see error output for missing modules or assets.

## Troubleshooting

- C# build errors: check target framework in `.csproj` and ensure Visual Studio has the appropriate workloads installed.
- Missing assets: verify images and `textures/` files exist at the paths the XAML expects.
- Python errors: run the Python script from a terminal and inspect tracebacks to locate missing modules or files.
