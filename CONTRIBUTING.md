Thanks for your interest in contributing to Roshan System! To keep contributions smooth and consistent, please follow these guidelines.

How to contribute
- Fork the repository and create a feature branch: git checkout -b my-feature
- Keep changes focused and small; one logical change per PR.
- Write clear commit messages and a descriptive PR title/body.
- If the change affects the build process or requires new dependencies, document them in README or add a requirements file.

Code style
- C# (WPF): follow standard .NET naming conventions. Add comments for non-obvious logic.
- Python: use PEP 8 style. Run a linter (flake8 or similar) before submitting.

Testing
- Add tests for new functionality where possible.
- Ensure existing behavior remains unchanged.

Pull requests
- Include a short description of the change and why it’s needed.
- Link to any related issues.
- If your PR adds or changes public behavior, update the README or CHANGELOG.md.

Communication
- Be respectful and constructive.
- If your change is large, open an issue first to discuss the approach.
